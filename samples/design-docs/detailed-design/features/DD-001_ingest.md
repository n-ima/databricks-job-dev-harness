# 詳細設計書: 入力取込（ingest）

| 項目 | 内容 |
|---|---|
| 設計ID | DD-001 |
| 版 | 1.0 / 承認日 2026-07-04（サンプル） |
| 上位設計 | BD-001 |

## 1. 機能概要

処理対象日の売上明細CSVをVolumeから読み込み、生データのまま `sales_raw` へ登録する。
型変換・検証は行わない（DD-002の責務）。

## 2. 入力

- ファイル: `/Volumes/<catalog>/<schema>/landing/input/sales_<YYYYMMDD>.csv`
  （YYYYMMDDは `processing_date` のハイフン除去）
- 形式: BD-001 4.1のとおり（UTF-8、ヘッダあり）

## 3. 出力

- テーブル: `<catalog>.<schema>.sales_raw`（BD-001 4.2。無ければ作成）
- 入力6列はすべて **string型のまま** 格納し、次のメタ列を付与する:
  `processing_date`（パラメータ値）, `source_file`（ファイル名）, `ingest_ts`（取込時刻UTC）

## 4. 処理仕様

1. `processing_date` パラメータを受け取り、`YYYY-MM-DD` 形式として妥当か確認する。
   不正なら異常終了（E-101）。
2. 入力ファイルの存在を確認する。存在しなければ異常終了（E-102）。
3. CSVをヘッダつきで読み込む（全列string）。列名がBD-001 4.1の6列と一致しない場合は
   異常終了（E-103）。
4. データ行が0件の場合は警告ログを出して**正常継続**する（0件登録）。
5. 冪等性: `sales_raw` から同一 `processing_date` の既存行をDELETEする。
6. 読み込んだ全行にメタ列を付与してINSERTする。
7. 登録件数をINFOログに出力する（値そのものは出力しない）。

## 5. エラー処理

| ID | 事象 | 動作 |
|---|---|---|
| E-101 | processing_date形式不正 | 異常終了。メッセージに受領値を含める |
| E-102 | 入力ファイル不存在 | 異常終了。メッセージに期待パスを含める |
| E-103 | ヘッダ列不一致 | 異常終了。期待列と実際の列を示す |

## 6. 受け入れ条件

| ID | 条件 | 確認方法 |
|---|---|---|
| AC-101 | 指定日のCSVの全データ行が `sales_raw` に登録される（件数一致、メタ列付与） | 結合テスト: 入力行数 = 登録行数 |
| AC-102 | 同一 `processing_date` で再実行しても行数・内容が変わらない（重複しない） | 結合テスト: 2回実行して件数一致 |
| AC-103 | 入力ファイル不存在時に異常終了し、メッセージに期待パスが含まれる | 結合テストまたは単体テスト |

## 7. 実装・テスト指針

- ファイルパス組み立て・ヘッダ検証・日付検証はSpark非依存の純粋関数として実装し、
  単体テスト（正常・境界・異常）を書く。Spark読み書きは薄いラッパに分離する。
