# 詳細設計書: レポート出力（export）

| 項目 | 内容 |
|---|---|
| 設計ID | DD-004 |
| 版 | 1.0 / 承認日 2026-07-04（サンプル） |
| 上位設計 | BD-001 / 前段: DD-003 |

## 1. 機能概要

`sales_daily_summary` の当日分をCSVレポートとしてVolumeへ出力する。

## 2. 入力

- `<catalog>.<schema>.sales_daily_summary` のうち `processing_date = パラメータ値` の行

## 3. 出力

- ファイル: `/Volumes/<catalog>/<schema>/landing/output/daily_sales_summary_<YYYYMMDD>.csv`
  （YYYYMMDDは処理対象日のハイフン除去）
- 形式: UTF-8、ヘッダあり、カンマ区切り、**単一ファイル**
- 列順: `processing_date,store_code,total_amount,total_quantity,detail_count`
- 並び順: `store_code` 昇順
- 日付は `YYYY-MM-DD` 形式で出力する

## 4. 処理仕様

1. 当日分の `sales_daily_summary` を読み込み、`store_code` 昇順に整列する。
2. CSV文字列を生成する（0行の場合も**ヘッダのみのファイルを出力**する）。
3. 冪等性: 同名ファイルが存在する場合は上書きする（part-xxxxx形式の複数ファイルや
   一時ディレクトリを残さない。単一ファイルとして書き出すこと）。
4. 出力パスと行数をINFOログに出力する。

## 5. エラー処理

| ID | 事象 | 動作 |
|---|---|---|
| E-401 | 出力先Volumeパスが存在しない | 異常終了。メッセージに期待パスを含める |

## 6. 受け入れ条件

| ID | 条件 | 確認方法 |
|---|---|---|
| AC-401 | 出力ファイルが生成され、行数 = 当日分summary行数 + 1（ヘッダ） | 結合テスト |
| AC-402 | 再実行しても出力先に同名の1ファイルのみ存在する（重複・残骸なし） | 結合テスト: 2回実行して確認 |
| AC-403 | データ行が `store_code` 昇順で、列順・日付形式が本書3節どおり | 単体+結合テスト |

## 7. 実装・テスト指針

- CSV文字列の生成（列順・並び・0行時ヘッダのみ・日付整形）はSpark非依存の
  純粋関数とし、単体テストで担保する。ファイル書き込みは薄いラッパに分離する。
