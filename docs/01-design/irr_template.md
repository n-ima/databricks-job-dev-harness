# 実装準備レビュー（Implementation Readiness Review: IRR）

## 目的

レビュー済み設計の内容を再審査するのではなく、このリポジトリの標準
（Databricks Job・Asset Bundles・CI/CD・AI支援開発）で安全に実装・試験・運用できる
状態かを確認する。手順は `.github/skills/design-readiness-review/SKILL.md` を参照。

## 判定

- [ ] `GO`: 必須項目がすべて確定し、重大な未解決事項がない
- [ ] `CONDITIONAL GO`: 期限・担当者付きの軽微な条件がある
- [ ] `NO-GO`: 実装の前提を変える未決事項がある

## 必須チェック

### 要求・境界

- [ ] 基本設計書と機能別・共通機能の詳細設計書が分離され、設計IDと承認版が一意
- [ ] 入力、出力、データ型、主キー、重複・欠損・遅延データの扱いが明確
- [ ] 業務ルールと受け入れ条件に一意なIDがある
- [ ] 正常系、異常系、再実行時の期待結果が明確
- [ ] 対象外と将来対応が区別されている

### Databricks適合性

- [ ] JobのタスクDAG、依存関係、パラメータ、タイムアウト、再試行方針が確定
- [ ] classic/serverless、Runtime、node type、autoscaling、policyの選択が確定
- [ ] 上記の選択が**実際のワークスペース制約と一致することを実機で確認した**
      （サーバレス専用ワークスペースにclassic cluster設計、等の前提崩れがない）
- [ ] dev/staging/prod の catalog、schema、外部接続先が分離されている
- [ ] Unity Catalog の所有者、実行主体、最小権限、行列制御の要否が確定
- [ ] データ量、SLA/SLO、コスト上限、同時実行、スケジュール、タイムゾーンが確定
- [ ] 冪等性、チェックポイント、部分失敗、再処理、バックフィルの方式が確定

### テスト可能性

- [ ] 業務ルールをSpark非依存の単体テストへ分離できる
- [ ] テストデータは合成または匿名化され、個人情報・機密情報を含まない
- [ ] 結合テスト用のschemaと実行権限がある
- [ ] データ品質、件数、重複、NULL、参照整合性の合格基準が数値化されている
- [ ] 性能試験が必要な場合、データ規模と合格閾値がある

### 運用・リリース

- [ ] 監視指標、通知先、Runbook、一次対応者、エスカレーション先が確定
- [ ] 本番リリースの承認者と変更可能時間帯が確定
- [ ] rollback（旧成果物再デプロイ）またはroll-forwardの判断基準が確定
- [ ] データ変更を伴う場合の復旧方法、保持期間、監査要件が確定

### AI利用

- [ ] CopilotおよびAI Dev Kitの利用が組織ポリシー・契約上許可されている
- [ ] AIへ送信禁止の情報区分が定義されている（environment.mdの機密区分）
- [ ] 生成コードは人が理解し、テストし、責任を持ってレビューする
- [ ] AI Dev Kit、MCP、skills、依存関係の採用バージョンが固定されている
