# 教訓ログ

このプロジェクトで得られた教訓を1行ずつ記録する。SessionStartフックが
「## 教訓」以下の箇条書き（先頭50行）を全セッションに自動注入するため、
書けば確実に効く。一般的なベストプラクティスはここではなくSkillに書く。

追記する条件（`.github/skills/harness-retrospective/SKILL.md` 参照）:

- ユーザーに同じ趣旨の訂正を受けたとき
- 同じアプローチで2回失敗したとき
- プロジェクト固有の暗黙知が判明したとき
- **コマンド・ツールの実行方法を試行錯誤の末に確立したとき**（成功したコマンドを
  そのまま書く。環境系の獲得知識は書かないと次のセッションで試行錯誤が再発する）

## 教訓

- [YYYY-MM-DD] （業務例: このワークスペースのdevカタログはXXチームの承認が必要）
- [YYYY-MM-DD] （環境例: devデプロイはCLIがPATHに無いため `python -m ...` で実行する）
